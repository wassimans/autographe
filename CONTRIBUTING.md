# About contributing to Autographe

Autographe is built by one person, by hand, in Rust. It's a craft project as much as a product, and the way it's made is part of what it is.

That shapes how I work with the people who use it.

**Bug reports are welcome.** If something is broken, please open an issue. Be specific about what you did, what you expected, and what happened instead. The more concrete, the more useful. I read every issue.

**Feedback is welcome.** If something feels wrong, awkward, or could be better, I want to hear it. Open an issue, or write to me directly. Even the smallest "this annoyed me" notes help.

**Feature requests are welcome, but most won't be built.** Autographe has a vision and a deliberate scope. Many good ideas don't belong in it. If you suggest a feature and I don't build it, that's not a rejection of you or the idea, it's a choice about what Autographe is.

**Pull requests are not accepted.** This is the part where I'm doing something less common, so I want to be clear about it.

I'm building Autographe alone, on purpose. Every line of code is written by hand, by me, in Doom Emacs. No AI assistance, no co-authors, no merged contributions. The work, the mistakes, the design choices, the slow learning, all of it is mine. That's not a temporary stage. It's the project.

I know this means some good code won't be written and some willing hands will have to find another project to help. I respect those hands, and I'm grateful for the offer. But Autographe is a solo craft, and keeping it that way is more important to me than going faster.

If you want to support the project, the most useful things you can do are:

- Use Autographe, and tell me how it goes.
- Open issues when something is wrong.
- Write about it if you like it.
- Tell other writers who might want it.

That's enough. That's more than enough.

Thank you for being here.

Wassim
